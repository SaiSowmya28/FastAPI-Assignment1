# 🛒 FastAPI E-Commerce API Simulation

## 📌 Project Overview

This project demonstrates the implementation of basic **FastAPI endpoints** using the **GET method** to simulate an e-commerce platform.

The APIs provide functionality to retrieve product information, categories, store inventory, stock availability, and promotional deals offered by the platform.

The goal of this assignment is to understand how **FastAPI handles API requests and responses**, along with practicing API route creation and response handling.

---

## 🛠 Technologies Used

- Python
- FastAPI
- REST API (GET Method)

---

## 📂 Implemented API Endpoints

### 1️⃣ E-Commerce Homepage API
Returns the basic welcome message or homepage information of the e-commerce platform.

Example error response: Welcome to E-commerce Platform API

---

### 2️⃣ Products API

Displays available products and their categories.

Example:
- Electronics
- Stationery
- Other product categories

---

### 3️⃣ Product Category Validation

Checks whether a requested category exists in the system.

If a category does not exist, an error message is returned.

Example error response: Category not found


---

### 4️⃣ Store Inventory API

Provides information about items stored in the store and their availability.

Features:
- Displays available items
- Shows inventory details

---

### 5️⃣ Stock Availability API

Checks whether a product is currently in stock.

Features:
- In stock / out of stock status
- Product availability tracking

---

### 6️⃣ Deals & Offers API

Displays promotional deals available on the e-commerce platform.

Examples:
- Best price deals
- Discounted products
- Special offers
